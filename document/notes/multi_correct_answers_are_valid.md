# 複数正解（2つ選べ等）は「誤り」ではなく仕様として扱う

## 結論
- `answer_result_text` が `正解は 1, 3 です。` のように **複数番号**を含むケースは、資格・設問によって **仕様として存在する**。
- そのため **「複数番号は間違いだから単一番号に矯正する」** という方針は取らない（過去問サイトの表示に従う）。

## 実務ルール（重要）
- `00_source` のバックフィル/再スクレイピングでは、サイト側が複数正解を返す場合は **そのまま保持**する。
- 検証や自動修正ロジックで「単一正解であるべき」という前提を置く場合は、必ず **対象資格・問題形式（例: true_false / group_choice など）を限定**する。

## 背景（よくある誤解）
- 二級建築士のように「5つの記述から正誤を選ぶ」形式では通常「正解は N です。」（単一）になるため、
  その経験則を他資格へ適用してしまうと誤判定になる。

## 備考
- このメモは、人間側の誤った指摘（“複数正解は存在しないはず”）を将来の作業で繰り返さないための一次情報。
