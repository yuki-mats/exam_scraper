# 二級土木施工管理技士試験 `category.json` 整備メモ

この文書は、`output/2nd-class-doboku-sekou/category/category.json` を新規作成・見直しする際に、どの粒度で分類するか、どの境界で分けるかを検討するための資料である。

## この文書の位置付け
- 日常の `questionSetId` 紐付けでは、完成済みの `category.json` の `name` / `description` / `matchingHints` を主根拠にする。
- この文書は、`category.json` 自体を整備する段階で
  - どこまで分けるか
  - どの論点を同じ `questionSetId` に入れるか
  - 何を `description` に書くか
  を決めるための検討メモである。

## 設計方針
- フォルダは試験の大章に対応させる。
- `questionSetId` は、受験者が誤答後に復習先を特定しやすい粒度にする。
- 教材目次をそのまま転記するのではなく、実際の設問本文で迷いにくい復習単位へ寄せる。
- `description` だけ読めば、どの問題を入れるべきか概ね判断できる状態を目指す。

## 境界ルール
### 工学基礎知識 vs 設計図・機械
- 地盤調査、土質、水理、構造力学は `工学基礎知識`
- 図面記号、構造物の部分名称、機械名称の読取りは `設計図・機械`
- ただし、特定工法の部材名称なら、その工法側を優先する
  - 例: 土留め部材名称は `基礎工`
  - 例: 海岸堤防の構造名称は `海岸・港湾`

### 土工 vs 施工計画
- 盛土材料、締固め、軟弱地盤対策、法面保護は `土工`
- 建設機械でも、施工計画全般や施工体制、コーン指数、作業能力は `施工計画`
- 個別作業と使用機械の対応を問うものは `土工` 側を優先する

### コンクリート vs 品質管理
- スランプ試験、配合、混和材料、打込み・締固めは `コンクリート`
- PDCA、管理図、ヒストグラム、レディーミクストコンクリートの品質管理手順は `品質管理`
- `試験` という語があっても、対象がコンクリート自体なら原則 `コンクリート`

### 構造物 vs 河川・砂防 vs 海岸・港湾
- 鋼材、橋梁、劣化機構は `構造物`
- 河川堤防、護岸、砂防えん堤、地すべりは `河川・砂防`
- 海岸堤防、消波工、浚渫、ケーソンは `海岸・港湾`

### 道路舗装 vs 施工管理法（基礎的な能力）
- 舗装の構造、施工、補修工法、破損は `道路舗装`
- 問題文が `施工管理法（基礎的な能力）` の枠で、同じ舗装テーマでも管理能力を問うなら `31_*`

### 法令系の切り分け
- 労働時間、賃金、就業規則は `労働基準法`
- 作業主任者、安全教育、危険防止措置は `労働安全衛生法` または `安全管理`
- 元方事業者、特定元方事業者、統括安全衛生責任者、安全衛生責任者、協議組織、作業間の連絡及び調整、作業場所の巡視は `13-3 元方事業者等の安全衛生管理体制`
- 建設業の許可、主任技術者、監理技術者は `建設業法`
- 道路占用や道路管理者は `道路関係法`
- 河川管理者の許可は `河川法`
- 法令知識ではなく現場安全対策そのものを問うなら `安全管理`

### 測量の切り分け
- 方位角、閉合比、測線長、閉合誤差などは `トラバース測量` または `トータルステーションによる測量`
- 後視・前視から器械高や地盤高を順に求める問題は `22-3 水準測量`

## 近年形式への対応
- 令和6年度以降のユニット問題は、`examLabel` のユニット名ではなく論点で分類する。
- `施工管理法（基礎的な能力）` は、施工計画・工程・安全・品質の再掲に見える問題でも `31_*` を優先する。
- `薬液注入` や `鋼構造物塗装` が `examLabel` に現れる回では、論点が既存 `questionSetId` に自然に寄るかを確認し、無理なら空IDで保留する。

## `description` に必ず入れたいこと
- 何を扱うカテゴリか
- どの近接論点とは分けるべきか
- このカテゴリに入りやすい代表語

## 目標状態
- `questionSetId` の紐付け担当者が、補助資料を読まなくても、`category.json` の `description` と `matchingHints` で大半を判断できる
- 補助資料は、`category.json` を更新する時だけ見ればよい
