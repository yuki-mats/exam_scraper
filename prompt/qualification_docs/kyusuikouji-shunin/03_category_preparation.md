# 給水装置工事主任技術者試験 `category.json` 整備メモ

この文書は、`output/kyusuikouji-shunin/category/category.json` を新規作成・見直しする際に、どの粒度で分類するか、どの境界で分けるかを検討するための資料である。

## この文書の位置付け
- 日常の `questionSetId` 紐付けでは、**完成済みの `category.json` の `name` / `description` / `matchingHints` を主根拠にする**。
- この文書は、`category.json` 自体を整備する段階で
  - どこまで分けるか
  - どの論点を同じ `questionSetId` に入れるか
  - 何を `description` に書くか
  を決めるための検討メモである。

## 設計方針
- フォルダは試験の大章に対応させる。
- `questionSetId` は、学習者が誤答後に復習先を特定しやすい粒度にする。
- 参考書の目次をそのまま転記するのではなく、**過去問の分類に使いやすい復習単位**に調整する。
- `description` だけ読めば、どの問題を入れるべきか概ね判断できる状態を目指す。

## 境界ルール
### 水道行政 vs 給水装置工事事務論
- 法令の条文要件、供給規程、水道事業者の業務は `水道行政`
- 主任技術者の責務、技能者配置、記録保存、建設業法、労働安全衛生は `給水装置工事事務論`

### 給水装置工事法 vs 施工管理
- 工法・設置位置・検査など、施工対象そのものは `給水装置工事法`
- 工程、品質、安全、道路復旧など管理の論点は `施工管理`

### 構造及び性能 vs 概要
- 基準、性能、システム安全が主題なら `給水装置の構造及び性能`
- 名称、用途、器具の種類、故障と対策が主題なら `給水装置の概要`

### 計画論 vs 工事法
- 方式選定、管径決定、水理計算、図面は `給水装置計画論`
- 実際の施工方法、埋設、設置位置、接合は `給水装置工事法`

## `description` に必ず入れたいこと
- 何を扱うカテゴリか
- どの近接論点とは分けるべきか
- このカテゴリに入りやすい代表語

- 補助資料は、`category.json` を更新する時だけ見ればよい
