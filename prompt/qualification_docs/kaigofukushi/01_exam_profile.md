# 介護福祉士試験 試験プロフィール

この文書は、介護福祉士試験の章立て、出題傾向、問題形式を整理し、解説文作成や `category.json` 検討の共通前提とするための資料である。

## 試験全体の見取り図
- 介護福祉士試験は、制度・権利・心理・疾患理解・生活支援・介護過程を横断して問う総合型試験である。
- `output/kaigofukushi/questions_json/2008-2024/20_merged_1` を見ると、問題形式は概ね次の比率で構成される。
  - `true_false`: 1956問
  - `flash_card`: 121問
  - `group_choice`: 1問
- 設問意図は `select_correct` が大半で、`select_incorrect` は少数である。
- したがって、学習者に有益な解説は、単なる正答確認ではなく、**似た概念をどう切り分けるか**、**事例で何を優先して判断するか**を短く明示する形になる。

## 旧課程との対応
2008年から2010年には旧科目名が混在する。解説文を書くときは、現行の学習テーマに読み替えて整理すると学習効率が良い。

- `（旧）社会福祉概論`、`（旧）老人福祉論`、`（旧）障害者福祉論`
  - 現行では主に `社会の理解` に接続する
- `（旧）社会福祉援助技術`
  - 現行では `介護過程`、`コミュニケーション技術`、`介護の基本` に接続する
- `（旧）介護概論`
  - 現行では `介護の基本` に近い
- `（旧）介護技術`、`（旧）形態別介護技術`
  - 現行では `生活支援技術` に近い
- `（旧）老人・障害者の心理`
  - 現行では `人間関係とコミュニケーション`、`認知症の理解`、`障害の理解` にまたがる
- `（旧）医学一般`、`（旧）精神保健`、`（旧）リハビリテーション論`
  - 現行では `こころとからだのしくみ`、`発達と老化の理解`、`障害の理解`、`医療的ケア` に接続する
- `（旧）家政学概論`
  - 現行では `生活支援技術` に接続する
- `（旧）レクリエーション活動援助法`
  - 現行では `生活支援技術`、`コミュニケーション技術` に接続する

## 科目群ごとの性格
### 人間の尊厳と自立 / 人間関係とコミュニケーション
- 権利擁護、自己決定、アドボカシー、自己覚知、ラポール形成などを扱う。
- 用語の定義暗記に見えても、実際は**何を尊重する場面か**、**何を援助者がしてはいけないか**が問われる。

### 社会の理解
- 社会保障制度、障害福祉制度、介護保険制度、家族・地域・雇用・権利擁護の制度知識を扱う。
- 頻出のひっかけは、**制度の目的**、**主体**、**対象者**、**年齢要件**、**給付内容**の取り違えである。

### 介護の基本 / コミュニケーション技術
- 利用者理解、尊厳保持、自立支援、チーム連携、観察、初期対応、声かけを扱う。
- 出題はケース型が多く、**良さそうに見えるが優先順位がずれている選択肢**を落とせるかが重要になる。

### 生活支援技術
- 食事、排泄、入浴、移動、整容、住環境、福祉用具、家事支援、レクリエーションを扱う。
- 単なる手順ではなく、**安全**、**自立支援**、**残存機能の活用**、**安楽**、**羞恥心への配慮**のどれを優先するかが問われやすい。

### 介護過程
- アセスメント、課題分析、目標設定、計画、実施、評価を扱う。
- 流れを機械的に覚えるより、**どの段階で何を確認するか**、**誰のニーズを基準にするか**を押さえる必要がある。

### 発達と老化の理解 / 認知症の理解 / 障害の理解
- 加齢変化、発達段階、認知症の症状と対応、障害特性と支援を扱う。
- 重要なのは、**症状の名称**よりも、**生活上どのような支障が出て、支援では何を調整するか**である。

### こころとからだのしくみ / 医療的ケア
- 解剖生理、疾患、観察、異常の見方、喀痰吸引、経管栄養などを扱う。
- 頻出のひっかけは、**症状と疾患の取り違え**、**観察項目の優先順位**、**実施条件と禁忌**である。

### 総合問題
- 複数科目をまたぐ事例統合が中心である。
- 学習者は、単発知識よりも、**利用者の状態像から支援方針を統合する**力を求められる。

## 問題形式の特徴
- 介護福祉士試験では、次の形式が多い。
  - 事例から最も適切な対応を選ぶ問題
  - 制度・概念の定義や対象を問う問題
  - 症状や身体機能から適切な観察・援助を問う問題
  - 組合せや空欄補充で複数知識を同時に確認する問題
- したがって、解説では次のいずれかを補うと学習効率が上がる。
  - 判断基準の優先順位
  - 類似概念との差
  - 制度の主語と対象
  - 症状と支援のつながり
  - 自立支援と安全配慮の両立の考え方

## 頻出の判定軸
- 本人意思を優先するか、家族・支援者の都合を優先していないか
- 自立支援になっているか、過介助や身体拘束に傾いていないか
- 権利擁護・尊厳保持の視点があるか
- その制度の主体、対象、要件が合っているか
- 症状・障害特性と援助内容が対応しているか
- 観察、アセスメント、計画、実施、評価の段階が逆転していないか

## 学習者がつまずきやすいポイント
- 「良いことを言っている」選択肢と、「この設問で最も適切」な選択肢を混同しやすい。
- 本人尊重を掲げながら、実際には家族や支援者の都合を優先している誤答を見抜きにくい。
- 安全確保を理由にした過介助や行動制限を、正しい支援だと誤認しやすい。
- 制度問題で、制度名は覚えていても、対象年齢・主体・給付内容が曖昧なまま解いてしまいやすい。
- 疾患や障害の名称は分かっても、生活支援へどうつなげるかが抜けやすい。

## 解説で「ハッとする」情報の方向
- その誤答がなぜ一見正しく見えるのかを指摘する。
- その上で、何を優先すべきだったかを明確にする。
- 似た概念や支援との違いを短く切り分ける。
- 現場での意味まで一段落とし込む。

## 解説文作成・カテゴリ設計への示唆
- 解説文では、科目名の復唱よりも、**なぜその判断軸が優先されるか**を書くと学習者の再利用性が高い。
- `category.json` では、科目名だけでなく、`制度理解`、`認知症ケア`、`移動介助`、`排泄支援` のように復習しやすい論点粒度へ落とす必要がある。
