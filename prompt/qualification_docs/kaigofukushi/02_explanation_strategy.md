# 介護福祉士試験 解説文戦略

この文書は、`03_prompt_add_explanationText.md` で介護福祉士試験の `explanationText` を書く際に、何をプラスアルファとして補うべきかを整理した補助資料である。

## 解説の狙い
- 介護福祉士の解説は、単なる正答解説で終わってはいけない。
- 常に次の3点を同時に狙う。
  1. 今回の問題の理解
  2. 介護福祉士試験らしい傾向やひっかけへの気づき
  3. 類題へ再利用できる判断軸の定着
- したがって、良い解説は「なぜこの肢が正しいか」だけでなく、「この資格では何が問われやすいか」「なぜその誤答に引っかかるのか」まで短く示す。

## 参照の基本方針
- 介護福祉士では、`20_merged_1` の `explanation_common_prefix` を主参照にする。
- `explanation_choice_snippets` は補助素材として使ってよいが、そのまま貼らず、`explanation_common_prefix` で示される判断軸に沿って統合する。
- `00_source` は参照しない。

## まず `qualification_docs` に整理しておくべきこと
介護福祉士で学習効率の高い解説を書くには、解説文そのものより前に、次の観点を資格別資料として固定しておく必要がある。

1. 頻出の判断軸
  - 本人意思、自立支援、安全配慮、権利擁護、制度主体、症状と支援の対応
2. 一見もっともらしい誤答パターン
  - 優しそうだが本人不在
  - 安全そうだが過介助
  - 丁寧そうだが設問主題からずれる
3. 似た概念の切り分け方
  - `アドボカシー` と `インフォームド・コンセント`
  - `自己覚知` と `自己開示`
  - `受容` と `迎合`
4. 科目ごとに必ず補うべきプラスアルファ
  - 制度: 主体、対象、要件
  - 事例: 優先順位、観察、支援の意図
  - 身体・疾患: 生活への影響、観察、援助
5. 学習者が「ハッとする」情報の型
  - なぜその誤答が魅力的に見えるのか
  - どの前提を取り違えると誤るのか
  - 類題で再利用できる判断軸は何か
6. その資格らしい出題の癖
  - 本人意思より家族意向に流れやすい
  - 安全を理由に過介助へ寄りやすい
  - 良い支援一般と設問主題を混同しやすい

## この試験の解説で最優先にすべきこと
- 単に `正しい。` `間違い。` と書かず、**なぜその場面ではその判断が優先されるのか**を示す。
- 特に次のどれで判定したかを明示すると学習効果が高い。
  - 本人意思の尊重
  - 自立支援と安全配慮のバランス
  - 権利擁護と尊厳保持
  - 制度の主体・対象・要件
  - 症状と支援内容の対応
  - 介護過程のどの段階か

## 介護福祉士で頻出のひっかけ
- 本人より家族の意向を優先している
- 安全を理由に過介助・行動制限へ寄り過ぎている
- 良い支援一般と、その設問で問われた概念を混同している
  - 例: `アドボカシー` と `インフォームド・コンセント`
  - 例: `自己覚知` と `自己開示`
  - 例: `自立支援` と `単なる見守り放棄`
- 制度の主体や年齢要件を取り違えている
- 症状名は正しいが、対応が合っていない
- 観察より先に決めつけた対応をしている

## 介護福祉士で「ハッとする」解説の条件
- 受験者が迷った理由を先回りして言語化している。
  - 例: 「安全に見えるが、残存機能を奪っている」
  - 例: 「丁寧に見えるが、本人の意思を代弁していない」
- 正答肢の良さを、抽象語ではなく判断軸で説明している。
  - 例: `本人の訴えを受け止めている`
  - 例: `観察を先に置いている`
  - 例: `家族ではなく本人を中心にしている`
- 誤答肢では、どこを直せば正しくなるかまで示している。
- その設問だけで終わらず、類題にも使える見分け方が残る。
- その資格で繰り返し現れる「ありがちなズレ方」が見える。

## 介護福祉士で避けるべき凡庸な補足
- 「尊重が大切」
- 「安全に配慮する」
- 「制度を理解しておく」
- 上のような一般論だけで終わる補足は弱い。
- 必ず、**何を尊重するのか**、**何から何を守るのか**、**誰にどう適用される制度か**まで一段具体化する。

## 介護福祉士の解説で意識する書き分け
### 1. 概念問題
- 用語の定義だけで終わらせず、**何と区別すべき概念か**を書く。
- 似た語が並ぶ場合は、誤答肢を正しい語に言い換えて差を明確にする。

例:
```json
"間違い。\n\n自己覚知は、自分の感情や価値観、偏りを客観的に理解することであり、自分について相手に話す自己開示とは異なる。対人援助では、この区別を押さえることが重要である。"
```

### 2. 制度問題
- 主語、対象、年齢、給付内容、設置主体のどこが判定点かを明示する。
- `制度名が分かる` だけでは弱く、**誰に何が適用される制度か**まで言い切る。

### 3. 事例問題
- まず、利用者に何が起きているかを短く要約する。
- 次に、その場面で優先すべき視点を示す。
  - 本人の訴えの受け止め
  - 状態変化の観察
  - 自立支援
  - 安全確保
  - 多職種連携
- その後で、他の選択肢がなぜずれるのかを一言で示す。
- 可能であれば、「その誤答がなぜ一見よく見えるのか」も短く添える。
  - 例: 自立支援に見えるが、実際は本人の状態変化を無視している
  - 例: 家族支援に見えるが、本人の自己決定を後景化している

### 4. 身体・疾患・医療的ケア問題
- 症状、観察項目、危険徴候、援助の方向性を結び付けて書く。
- 単に疾患名の説明に寄らず、**介護場面で何に注意するか**へ落とす。

### 5. 生活支援技術問題
- 正しい手順だけでなく、その手順が必要な理由を書く。
- 特に次を優先して言語化すると学習効率が良い。
  - 残存機能の活用
  - 安全確保
  - 安楽
  - 羞恥心や尊厳への配慮
  - 介助量の適切さ

## `questionIntent` と介護福祉士らしい解説
- `select_correct` が多いが、介護福祉士では誤答肢にも「一般論として完全な誤りではないが、この場面の主題ではない」ものが含まれる。
- その場合も、解説は `間違い。` で始めたうえで、**この設問の判断軸に照らすとずれている理由**を書く。
- 逆に、正答肢では「正しい。」のあとに、**何を優先したから適切なのか**を具体化する。

例:
```json
"間違い。\n\n利用前に説明して同意を得ること自体は重要だが、この設問で問われているのは本人の声を代弁して実現につなげるアドボカシーの視点である。インフォームド・コンセントとの違いを区別して押さえる必要がある。"
```

## 科目群ごとの補足知識
### 人間の尊厳と自立 / 人間関係とコミュニケーション
- `本人の主体性`、`自己決定`、`権利擁護` を優先して書く
- 似た概念との差を短く入れる

### 社会の理解
- 制度の主語、対象、要件、給付を整理して書く
- 年齢要件、設置主体、申請主体の取り違えを正す

### 介護の基本 / コミュニケーション技術
- 観察、受容、共感、信頼関係形成、多職種連携のどれが軸かを明示する
- 実習指導や初期対応では、答えを与えるより気づきを促す助言かどうかも判定軸になる

### 生活支援技術
- 手順だけでなく、目的を書く
- `安全だから何でも制限する` 方向へ流れていないかを確認する

### 介護過程
- アセスメント、目標設定、実施、評価のどの段階かを明示する
- 利用者ニーズと支援者都合を区別する

### 発達と老化の理解 / 認知症の理解 / 障害の理解
- 症状・障害特性と、生活上の支援を結び付ける
- 病名の説明で終わらず、介護実践上の意味まで一文足す

### こころとからだのしくみ / 医療的ケア
- 異常の兆候、観察項目、危険性、介護職が取るべき対応を整理する
- 医療的ケアは実施条件、観察、異常時対応の順で書くと整理しやすい

### 総合問題
- 単発知識の正誤ではなく、状態像を統合して最も優先すべき支援を示す
- 複数の選択肢が一見よく見える場合は、優先順位の理由を書く

## 介護福祉士で避けるべき悪い解説
- `選択肢2：正しい。`
- `記述のとおり。`
- `本人を尊重することが大切。` だけで具体性がない
- `安全のためである。` だけで、何が危険で何を防ぐかがない
- `制度上そうなっている。` だけで、主体や対象が書かれていない
- `答えは4` の繰り返しで終わる

## 介護福祉士の学習効率を上げる解説の型
- 1文目:
  - `正しい。` または `間違い。`
- 2文目:
  - 判定の直接理由
- 3文目:
  - 類題にも使える判断軸
- 4文目が必要なら:
  - 「なぜ迷いやすいか」または「何と混同しやすいか」

例:
```json
"間違い。\n\n誤っているのは、家族の意向を本人の意思より優先している点である。介護福祉士は、家族支援も行うが、まず本人の意思と状態を基準に支援を組み立てる。事例問題では、誰を中心に考える場面かを見失わないことが重要である。"
```
